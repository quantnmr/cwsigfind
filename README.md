# CWSigFind

**Live aggregator for CW (Morse) amateur-radio spots — POTA, SOTA, WWFF, BOTA, DX cluster, RBN — with click-to-tune Hamlib radio control.**

> 🟢 **Try it in your browser, no install:** [**quantnmr.github.io/cwsigfind**](https://quantnmr.github.io/cwsigfind/)

CWSigFind merges multiple real-time CW spot feeds into a single sortable, filterable table. Click a row and your radio QSYs to that frequency and mode. There are two flavors:

- A **browser-only lite demo** (the link above) covering POTA / SOTA / WWFF / BOTA + an NCDXF beacon widget. Zero install.
- A **full async Python daemon** that adds DX clusters, the Reverse Beacon Network, IOTA detection from cluster comments, WCA/DCI/DFCF castle-award detection, and Hamlib `rigctld` integration so click-a-spot tunes a real radio.

Both are in this one repo.

---

## Two flavors

### Lite — the browser demo

Polls four CORS-friendly feeds directly from your browser:

| Source | Endpoint | Cadence |
|---|---|---|
| **POTA** | `api.pota.app/spot/activator` | 30s |
| **SOTA** | `api2.sota.org.uk/api/spots/-1/all` | 30s |
| **WWFF** | `spots.wwff.co/static/spots.json` | 30s |
| **WWBOTA** | `api.wwbota.org/spots/?age=1` | 60s |

Plus a deterministic **NCDXF / IARU beacon widget** (bottom-left) that recomputes the 18-station × 5-band schedule locally every 10s.

**Limitations of the lite version:**

- ❌ No DX cluster (telnet — can't be reached from a browser).
- ❌ No RBN (also telnet).
- ❌ No IOTA detection — IOTA refs live in cluster comments, and there's no cluster in the lite build. (The IOTA group-name catalog ships with the lite demo anyway, primed for any feed that ever emits an IOTA-tagged spot, and refreshed weekly by a GitHub Action.)
- ❌ No radio control — clicking a row shows a friendly toast pointing you here.

### Full — the Python daemon

Everything in the lite version, plus:

- **DX cluster** over telnet, with **multiple simultaneous nodes** supervised independently (defaults: NC7J for general traffic, S50CLX pre-filtered to POTA/SOTA/WWFF/BOTA/HEMA tags, DXMAPS configured but off).
- **Reverse Beacon Network** — the CW skimmer firehose. Perfect for CWT.
- **IOTA detection** from cluster comments (`NA-052`, `EU-005`, …) with island-name lookup from a weekly-refreshed `~/.cwsigfind/iota_groups.json` cache.
- **WCA / DCI / DFCF** castle-award detection from cluster comments.
- **Hamlib radio control** via `rigctld` — click a spot and your radio QSYs. ~300 supported rigs, persistent rig config in `~/.cwsigfind/rig.json`, auto-reconnect on startup.

The daemon's UI is identical in spirit to the lite demo, served over WebSocket from a local FastAPI app at `http://127.0.0.1:8765`.

## Lite demo — how it works

The lite demo lives entirely under `docs/` and is served by GitHub Pages from `main`:

```
docs/
  index.html         entry point; loads app.js as an ES module
  styles.css         dark theme, lifted from the full app's UI
  app.js             orchestrator — wires chips, table, beacons, help drawer
  lib/
    spot.js          band map (IARU edges) + dedup-key shape
    geo.js           parse_location_desc + longest-prefix country lookup
    beacons.js       NCDXF schedule (18 stations × 10s × 5 bands)
    sources.js       four pollers with exponential backoff and onSpot push
    prefixes.js      autogenerated from src/cwsigfind/geo.py
  iota_groups.json   1178-row IOTA catalog, same-origin (no CORS upstream)
```

There's no build step, no `node_modules`. The browser loads `app.js` as `<script type="module">` and pulls in the rest via ES imports.

**CORS, as verified live during build:**

| Endpoint | `Access-Control-Allow-Origin` |
|---|---|
| `api.pota.app` | `*` |
| `api2.sota.org.uk` | `*` |
| `spots.wwff.co` | `*` |
| `api.wwbota.org` | reflects the `Origin` header |
| `iota-world.org` | **none** — that's why we ship the catalog same-origin and refresh it via [a weekly GitHub Action](.github/workflows/iota-refresh.yml) |

The four pollers each manage their own dedup ring (capped at 5000 keys per source), so a flaky upstream backs off (exponential, 5-minute cap) without blocking the others.

The callsign-prefix → country table that `lib/geo.js` uses is auto-generated from the canonical Python table in `src/cwsigfind/geo.py`, so the two implementations can never drift apart in production:

```bash
python3 scripts/gen_prefixes.py
# writes docs/lib/prefixes.js — commit the result
```

## Full Python daemon

### Prerequisites

- **Python 3.11+**
- **[uv](https://github.com/astral-sh/uv)** — fast Python package manager. Install it once:

  ```bash
  # macOS / Linux:
  curl -LsSf https://astral.sh/uv/install.sh | sh
  ```

- **Hamlib** (optional — only needed if you want radio control):

  ```bash
  # macOS:
  brew install hamlib

  # Debian / Ubuntu:
  sudo apt install libhamlib-utils
  ```

### Install + run

```bash
git clone https://github.com/quantnmr/cwsigfind
cd cwsigfind
uv sync
cp config.example.toml config.toml
# Edit config.toml — at minimum put your callsign in for DX cluster / RBN logins.
uv run cwsigfind --config config.toml
# Then open http://127.0.0.1:8765
```

If you'd rather not use uv:

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
cwsigfind --config config.toml
```

### Configuration reference

Most knobs have sensible defaults. The full schema lives in [`config.example.toml`](config.example.toml); the headline sections:

| Section | Key | Default | What it does |
|---|---|---|---|
| (top) | `callsign` | `"N0CALL"` | Your amateur callsign — used for DX cluster / RBN logins. |
| `[filter]` | `modes` | `["CW"]` | Server-side mode filter; spots not matching are dropped before the WebSocket. |
| `[filter]` | `bands` | `[]` | Band names like `["40m", "20m"]`. Empty = all bands. |
| `[filter]` | `regions` | `[]` | Python regex list matched against POTA `locationDesc` (e.g. `["^US-"]`). |
| `[filter]` | `callsign_prefixes` | `[]` | Restrict to specific callsign prefixes. |
| `[sources.pota]` | `poll_interval_seconds` | `30` | POTA poll cadence. |
| `[sources.sota]` | `poll_interval_seconds` | `60` | SOTA poll cadence. |
| `[sources.wwff]` | `poll_interval_seconds` | `30` | WWFF poll cadence (**never below 30** — upstream is shared). |
| `[sources.wwbota]` | `poll_interval_seconds` | `60` | WWBOTA poll cadence (**never below 60**). |
| `[[sources.dxcluster]]` | `name`, `host`, `port`, `login_commands` | NC7J + S50CLX preset | One block per cluster node — any number, supervised independently. |
| `[sources.rbn]` | `enabled` | `false` | RBN is a firehose; flip on once your band filter is right. |
| `[web]` | `host`, `port` | `127.0.0.1` / `8765` | Where to serve the UI. |
| `[rig]` | `auto_connect` | `true` | Reconnect to the saved rig on startup. |

### Radio control (full only)

The daemon drives your rig via Hamlib's `rigctld`. Open the **Radio…** drawer in the top-right and:

- **Model** — dropdown auto-populated from `rigctl --list` (~300 supported rigs)
- **Port** — dropdown from `pyserial`'s enumeration. Pick **Custom…** for network rigs (`192.168.1.50:4533`) or any device path that wasn't enumerated.
- **Baud** — optional; Hamlib picks a sensible default if blank.
- **rigctld TCP listen port** — only change if `4532` clashes.

Click **Connect & Test**. The daemon spawns `rigctld -m <model> -r <port> -t <listen_port>`, opens a TCP socket, and asks the rig for its current VFO + mode to prove the round-trip works. On success the config is persisted to `~/.cwsigfind/rig.json` and reconnected automatically on the next startup (toggle `[rig].auto_connect = false` if you'd rather click Connect each time).

Once connected, **every spot row is click-to-tune** — the daemon sends `F <freq_hz>` and `M <mode>` to rigctld. The footer pill stays green and shows the rig's current VFO; it re-reads every 5s so VFO changes you make on the rig are reflected in the UI.

Test without a real radio by selecting **Hamlib / Dummy** (model #1) with any port (e.g. `/dev/null`).

### CWT preset (CWops mini-tests)

The CWops CWT sprints are 1-hour CW contests held weekly:

| Day | UTC |
|---|---|
| Wed | 1300Z (US "lunch") |
| Wed | 1900Z (EU evening + US morning) |
| Thu | 0300Z (Asia / US west-coast night) |
| Thu | 0700Z (Asia / EU early-morning) |

To use cwsigfind as a CWT board:

1. Set `[sources.rbn] enabled = true` in `config.toml` — RBN is the spot firehose for CWT.
2. Confirm `[filter].modes = ["CW"]` (the default).
3. Optionally narrow the band filter — CWT calling frequencies cluster ±5 kHz of these (kHz):

   | Band | Freq |
   |---|---|
   | 160m | 1818 |
   | 80m  | 3528 |
   | 40m  | 7028 |
   | 30m  | 10118 |
   | 20m  | 14028 |
   | 17m  | 18078 |
   | 15m  | 21028 |
   | 12m  | 24908 |
   | 10m  | 28028 |

No code change is needed — the existing RBN source and band filter together cover the use case.

### RBN tips

RBN volume varies wildly: quiet on a flat Sunday afternoon, deafening during CWT. The combination that works for most operators is:

- `[filter].modes = ["CW"]` (keep RBN's RTTY skimmers out)
- A 1-3 band selection via the UI band chips (you can persist this in `[filter].bands` once you've found your preferred set)
- An optional region filter — `regions = ["^W", "^K", "^N"]` for North America-only RBN reception, etc.

## Architecture

```
  ┌────────────┐  HTTP   ┌──────────────┐
  │  POTA API  │◄────────│  POTASource  │┐
  └────────────┘         └──────────────┘│
  ┌────────────┐  HTTP   ┌──────────────┐│
  │  SOTA API  │◄────────│  SOTASource  │┼┐
  └────────────┘         └──────────────┘││
  ┌────────────┐  HTTP   ┌──────────────┐││   ┌────────┐   ┌──────────────┐
  │  WWFF API  │◄────────│  WWFFSource  │┼┼──►│SpotBus │──►│ FastAPI / WS │──► browser
  └────────────┘         └──────────────┘││   └────────┘   └──────────────┘
  ┌────────────┐  HTTP   ┌──────────────┐││       ▲
  │ WWBOTA API │◄────────│ WWBOTASource │┘│       │
  └────────────┘         └──────────────┘ │   SpotStore (ring buffer + dedup)
  ┌────────────┐ telnet  ┌──────────────┐ │
  │ DX cluster │◄────────│DXClusterSrc  │─┘
  └────────────┘         └──────────────┘
  ┌────────────┐ telnet  ┌──────────────┐
  │    RBN     │◄────────│   RBNSource  │──┐
  └────────────┘         └──────────────┘  │
                                           ▼
                                       SpotBus
```

- `SpotSource.start()` wraps `run()` in a supervisor that restarts on exceptions with exponential backoff up to 60s.
- `SpotBus` is fan-out async pub/sub with a bounded queue per subscriber and drop-oldest on overflow.
- `SpotStore` is the dedup gate; if a spot's `dedup_key` is already known it's silently dropped. Last 2000 spots are kept in a ring buffer; the 5000-key LRU dedup set covers the same horizon comfortably.

### What "new spot" means here

- **POTA / SOTA / WWFF**: each upstream provides a stable numeric ID (`spotId`, `id`); we dedup on that.
- **WWBOTA**: no numeric ID, so we build a stable key from `call:type:time:freq`.
- **DX cluster / RBN**: no native ID, so we bucket by `(source, callsign, kHz_rounded, minute)` — the same callsign re-spotted on the same kHz inside one minute is treated as a single spot.

The store keeps the last 2000 spots in memory and a 5000-key LRU for dedup. The web UI keeps the most recent 1000 client-side.
